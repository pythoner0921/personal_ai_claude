# Personal AI Memory Engine v0.1.0

**Initial release** — A persistent memory system for Claude Code that learns user preferences from natural interaction and reinjects them as context in every session.

## Release Summary

This release delivers a complete, self-contained memory engine built on Claude Code hooks. The system captures behavioral signals, extracts preferences, manages their lifecycle, ranks them by relevance, and injects personalized context — all automatically, with no user configuration required.

## Core Features

### Preference Extraction & Storage

- Keyword-based marker detection from user prompts and assistant responses
- Optional LLM-based behavior analysis via local Ollama (qwen2.5:3b)
- YAML storage for preferences, JSONL for raw events and audit logs
- No external database or cloud service required

### Memory Lifecycle

Preferences evolve through a four-state machine:

```
candidate → recent → stable → archived
```

| State | Criteria | Storage |
|-------|----------|---------|
| candidate | Newly extracted, evidence < 2 | `inference/preference_candidates.jsonl` |
| recent | evidence ≥ 2, confidence ≥ 0.50 | `memory/recent_tendencies.yaml` |
| stable | evidence ≥ 5, confidence ≥ 0.75, stability ≥ 0.60 | `memory/stable_preferences.yaml` |
| archived | confidence < 0.3 OR last_seen > 90 days | `memory/archive.yaml` |

Archived preferences are never deleted — they move to `archive.yaml` for potential recovery.

### Ranking Formula

Every preference receives a composite score to determine injection priority:

```
score = priority × 3 + confidence × decay + relevance + scope + affinity
```

- **priority**: stable (3) outranks recent (2), multiplied by 3
- **confidence**: 0.0–0.95, computed from evidence frequency and cross-context stability
- **decay**: `0.97 ^ days_since_last_seen` — naturally deprioritizes stale preferences (~23-day half-life)
- **relevance**: keyword overlap between preference description and current query
- **scope**: +0.2 bonus when preference scope matches current project
- **affinity**: +0.2 per task-type match (max +0.4)

Top 4 preferences (within a 600-character budget) are injected into each prompt.

### Task-Aware Retrieval

A rule-based classifier identifies 7 task types (debugging, coding, architecture, explanation, configuration, documentation, review) and applies affinity bonuses:

- Debugging prompts boost "concise" and "evidence" preferences
- Architecture prompts boost "summary" and "table" preferences
- Coding prompts boost "modular" and "compact" preferences

### Reflection Engine

Synthesizes higher-level user profiles from accumulated preferences:

- Groups preferences by theme (communication style, code approach, general)
- Generates 1-sentence summaries per theme
- Triggers every 50 captured events or at session start
- Injected before individual preferences for quick context

Example output:
> "Communication: lead with summaries, use tables for comparisons, keep concise. (avg confidence 83%)"

### Memory Quality Control

- **Synonym deduplication**: Maps known variants to canonical forms (e.g., "dislikes verbose output" → "prefers concise output")
- **Fuzzy merge**: Jaccard similarity with semantic token normalization (short→concise, responses→output, incremental→modular). Threshold: 0.6
- **Noise filtering**: Keyword blocklist removes task-specific patterns that aren't real preferences
- **Deprecated deduplication**: Prevents accumulation of duplicate entries in deprecated storage

### Memory Health Monitoring

Runs at every session end and reports:

- Total active preferences (stable + recent)
- Archived and deprecated counts
- Average confidence score
- Duplicate candidate detection
- Usage statistics (used vs. unused preferences)
- Low-decay warnings

## Tooling

### CLI Interface

```bash
python cli.py memory health          # Health report
python cli.py memory list            # List active preferences with scores
python cli.py memory search <query>  # Search by keyword
python cli.py memory reflect         # Generate reflections
python cli.py memory consolidate     # Run dedup + archival
python cli.py memory dashboard       # Generate HTML dashboard
python cli.py memory export <file>   # Export to JSON
python cli.py memory import <file>   # Import from JSON (with backup)
```

### API Layer

```python
from engine.api import get_active_memories, search_memories, get_memory_health_text

memories = get_active_memories()          # All active preferences with decay scores
results = search_memories("concise")      # Keyword search
print(get_memory_health_text())           # Formatted health report
```

### HTML Dashboard

Self-contained sortable dashboard with stat cards, reflection summaries, and a preference table with confidence/decay bars.

### Export / Import

- Export bundles all YAML files into a single versioned JSON
- Import restores from JSON, automatically backing up existing files
- Format version: 1.0

### Test Suite

39 tests covering:

- Lifecycle transitions and archival rules (10 tests)
- Ranking score ordering, decay, scope, and task affinity (16 tests)
- Synonym merge, fuzzy merge, tokenization, noise filtering, field preservation (13 tests)

## Why This Project Matters

Most AI tools treat every conversation as a blank slate. The Personal AI Memory Engine is a step toward AI that genuinely learns from working with you — not through fine-tuning or cloud sync, but through a lightweight, local, transparent memory layer that you own and control.

Every preference is stored in plain YAML files you can read, edit, or delete. Every decision the system makes is logged and auditable. The engine adds intelligence without removing agency.

## What's Next

Potential directions for future development:

- **Cross-project memory sharing** — Propagate global preferences across multiple project contexts
- **Embedding-based similarity** — Replace rule-based fuzzy merge with lightweight local embeddings for higher-accuracy deduplication
- **Preference conflict resolution** — Detect and resolve contradictory preferences (e.g., "prefers detailed explanations" vs. "prefers concise output")
- **Multi-user support** — Separate preference profiles for different team members in shared projects
- **Memory visualization** — Interactive timeline showing preference evolution, confidence changes, and lifecycle transitions
